- `env/.env`

  - 增加 `VITE_SERVER_BASEURL__WEIXIN_DEVELOP`、`VITE_SERVER_BASEURL__WEIXIN_TRIAL`、`VITE_SERVER_BASEURL__WEIXIN_RELEASE` 注释示例。
- `env/.env.development`、`env/.env.production`、`env/.env.test`

- `src/env.d.ts`

- `src/utils/index.ts`
  - 删除原本硬编码的微信小程序 develop / trial / release 地址。
  - 改为按 `uni.getAccountInfoSync().miniProgram.envVersion` 从 `import.meta.env` 读取。
  - 未配置专用地址时回退 `VITE_SERVER_BASEURL`。
- `vite.config.ts`
  - 合并上游关于非 H5 端 dev 命令仍是 build command 的注释。

决策：只改配置来源，不改当前默认地址，避免影响本地和已有后端联调。

### 微信开发者工具 CLI path

对应：`d3a411d8`、`278610af`、`36595585`、`d1898f21`、`63aff76d`。

- `env/.env`
  - 增加 `WECHAT_DEVTOOLS_CLI_PATH` macOS / Windows 注释示例。
  - 使用非 `VITE_` 前缀，避免暴露给客户端。
- `vite.config.ts`
  - 增加 `envDir`。
  - 保留 `loadEnv(mode, envDir)` 读取客户端 `VITE_` 变量。
  - 增加 `loadEnv(mode, envDir, '')` 读取 `WECHAT_DEVTOOLS_CLI_PATH`。
  - 调用 `openDevTools({ mode, wechatDevtoolsCliPath: WECHAT_DEVTOOLS_CLI_PATH })`。
- `scripts/open-dev-tools.js`
  - `_openDevTools(env, options)` 支持 `wechatDevtoolsCliPath`。
  - macOS / Windows 微信小程序优先使用配置路径，未配置时回退默认路径。
  - 打开失败时打印当前 CLI 命令和配置提示。

决策：不从 `process.env.WECHAT_DEVTOOLS_CLI_PATH` 直接读，统一由 Vite `loadEnv` 传入，和上游最终状态一致。

## 暂不合并

### tabbar 同步专项

对应：`03e115f6`、`f8cb3252`、`1c1c7bd3`、`eab2e808`、`bbf56546`、`ce8ed973`。

结构差异和风险：

- 上游 `src/tabbar/store.ts` 已把 `tabbarList` 从 `reactive` 数组改成角色过滤的 `computed`，代码大量使用 `tabbarList.value`；当前项目仍是普通 `reactive` 数组。
- 上游 `setCurIdx` 没有当前项目的登录态和白名单/黑名单 guard；当前项目的 `tokenStore.hasLogin`、`isNeedLoginMode`、`judgeIsExcludePath` 必须保留。
- 上游角色 tabbar 已经和最终 store 结构耦合；本轮只抽同步逻辑，不引入角色 tabbar。
- 上游 `src/tabbar/index.vue` 已抽出 `TabbarItem.vue`，当前项目仍是单文件实现；因此是重写/移植思路，不是套 diff。
- `03e115f6` 中的 router interceptor 改动在 `packages/cli/features/login/files/src/router/interceptor.ts`，是 CLI 模板文件，不是当前 runtime。

后续专项提取思路：

- `src/tabbar/store.ts`
  - 新增 `normalizeRoutePath(path?: string)`。
  - `isPageTabbar()` 使用 normalize 后路径，并把 `/` 视为首页 tabbar。
  - 新增 `findTabbarIndexByPath()` 和从页面栈倒序寻找最近 tabbar 页的逻辑。
  - `setAutoCurIdx()` 找不到当前路径时，回退到页面栈里的最近 tabbar。
  - 新增 `syncCurIdxByCurrentPage()` / `syncCurIdxByCurrentPageAsync()`。
  - 新增 `isCurrentRouteTabbarItem(index)`。
- `src/tabbar/index.vue`
  - 点击当前 tab 时，只有“当前高亮且当前路由就是该 tab”才阻止跳转。
  - `switchTab` / `navigateTo` 完成后调用同步。
- `src/App.vue`、`src/App.ku.vue`
  - `onShow` 后同步当前 tabbar index。
  - H5 下监听 `visibilitychange` / `pageshow` 后同步 tabbar。

专项验证：

- H5：工作台、审批、通讯录、消息、我的之间连续切换，高亮不乱。
- H5：进入非 tabbar 子页面后返回，高亮恢复到最近的 tabbar 页。
- H5：浏览器 tab 切走再切回，高亮仍正确。
- 微信小程序：`switchTab` 后高亮正确；分享/直达路径进入时高亮正确。
- 未登录点击需要登录的 tab 时，当前登录拦截逻辑仍然生效。
